01. Theoretical Foundations

Data Persistence & State Management

Applications running purely in memory lose all state upon termination. To achieve Data Persistence, software requires a Database Management System (DBMS). Python supports numerous database paradigms, but relational databases remain the industry standard for structured, transactional data.

Types of Databases Used with Python

  • Relational (SQL): Data is stored in tables (rows/columns) with strict schemas. Examples: MySQL, PostgreSQL, SQLite. Ideal for transactional integrity (ACID compliance).
  • Non-Relational (NoSQL): Data is stored in flexible JSON-like documents or key-value pairs. Examples: MongoDB, Redis. Ideal for rapid scaling and unstructured data.

MySQL Database Connectivity with Python: Python utilizes the mysql-connector-python library (an implementation of the Python DB-API 2.0 specification) to establish a network bridge between the application logic and the SQL server engine.

02. Industry Context

CareData: Hospital Management Database System

A modern hospital generates massive amounts of structured data: patient demographics, doctor schedules, and appointment logs. If this data is stored in flat files (like CSVs), it leads to data duplication, concurrent write errors, and security breaches.

CareData Systems utilizes an SQL Relational Database (MySQL) connected to a Python backend. This ensures Referential Integrity—for example, an appointment cannot be booked for a patient ID that does not exist in the system.

03. Code Implementations

1 Establishing MySQL Connectivity

The fundamental step required to authenticate and connect Python to the MySQL server instance. Connection objects must be securely managed.

db_config.py 
import mysql.connector
from mysql.connector import Error

def create_connection():
    try:
        # Establishing connection via kwargs
        connection = mysql.connector.connect(
            host='localhost',
            database='hospital_db',
            user='admin_py',
            password='secure_pass_123'
        )
        if connection.is_connected():
            print("Successfully connected to MySQL database")
            return connection

    except Error as e:
        print(f"Error while connecting to MySQL: {e}")
        return None

2 Insert Data (C)

Using parameterized queries (%s) is critical to prevent SQL Injection attacks. The commit() function permanently saves the transaction.

insert_patient.py 
def add_patient(conn, name, dob):
    cursor = conn.cursor()

    # Parameterized SQL query
    query = """
        INSERT INTO Patients (first_name, dob)
        VALUES (%s, %s)
    """
    values = (name, dob)

    cursor.execute(query, values)
    conn.commit() # Save changes

    print(f"Inserted ID: {cursor.lastrowid}")
    cursor.close()

3 Fetch Data (R)

Executing a SELECT query returns a result set. The cursor's fetchall() method pulls this data into a Python list of tuples.

fetch_appointments.py 
def get_appointments(conn, doc_id):
    cursor = conn.cursor(dictionary=True)

    query = """
        SELECT appt_date, patient_id
        FROM Appointments
        WHERE doctor_id = %s
    """
    cursor.execute(query, (doc_id,))

    # Fetch all matching rows
    records = cursor.fetchall()

    for row in records:
        print(row['appt_date'])

    cursor.close()

04. Quality Assurance & Evaluation

Security / Architecture Issue Mitigation Protocol
SQL Injection (SQLi) Never use Python f-strings or string concatenation to build SQL queries. Always use parameterized queries (e.g., VALUES (%s)) allowing the connector to escape malicious input safely.
Connection Leaks Failing to close cursors and connections depletes the database server's connection pool. Always invoke cursor.close() and conn.close() in a finally block.
Uncommitted Transactions Executing an INSERT or UPDATE statement does not save it to the disk. You must explicitly call conn.commit() to persist changes.

Q1: Explain the role of a Database Cursor in Python.

A: A cursor is a temporary memory work area created in the system memory when an SQL statement is executed. In Python, the cursor object is used to execute statements to communicate with the MySQL database, and to fetch data from the result sets row by row.

Q2: What is Referential Integrity in Relational Databases?

A: It is a property that ensures relationships between tables remain consistent. It dictates that a Foreign Key value must match an existing Primary Key value. E.g., an appointment cannot reference a patient_id that has been deleted.
