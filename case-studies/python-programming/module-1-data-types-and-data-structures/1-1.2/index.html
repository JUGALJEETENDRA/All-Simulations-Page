02. Industry Case Study

Meltridge Retail Bank & The 07:03 Incident

Meltridge Retail Bank runs a nightly batch job that recalculates interest, service fees, and loyalty cashback for 2.3 million savings accounts. It leans entirely on native operators.

The Incident:

The job silently overcharged 8,412 customers by an average of ₹6.18 because one division operator was doing something the engineer didn't expect.

Root Cause: The fee routine used total_paise // 100 assuming floor division always rounds toward zero. For negative adjustment entries (refunds), floor division rounds toward negative infinity, silently shifting the result.

Code Implementation

def calculate_fee(balance_paise: int, pct: float, cap: int) -> int:
    # Multiply before dividing to preserve precision
    raw_fee = (balance_paise * pct) / 100
    
    # banker's rounding
    rounded_fee = round(raw_fee)          
    
    return min(rounded_fee, cap)

fee = calculate_fee(balance_paise=842_150, pct=1.25, cap=1500)
print(fee)  # Output: 1500 -> capped
