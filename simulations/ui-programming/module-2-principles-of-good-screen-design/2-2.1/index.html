Information Architecture Lab

Welcome to the interactive lab for UI Programming – Module 2.1: Information Architecture and Application Structure. Investigate how content categorization, sitemaps structuring, and flat hierarchy balances decrease student discovery friction.

1. Understand Sort pages and construct parent-child tree sitemaps
2. Organize Connect page nodes and repair poorly structured categories
3. Apply Critique structural defects and design portal taxonomies

📖 Learning Recap: Activity 1 (Basic Sorting)
What Usability Problems Were Found? Scattered, unclassified resources increase cognitive lookup fatigue.
Which Principle Was Learned? Semantic grouping and category categorization bounds.
How Can It Be Applied in Real Projects? Group similar tools under clear conceptual umbrellas (Academic, Fees, Support).

📖 Learning Recap: Activity 3 (Navigation Flow)
What Usability Problems Were Found? Long nested paths increase click counts, and require users to memorize navigation loops.
Which Principle Was Learned? Navigation click depth efficiency and cognitive user effort.
How Can It Be Applied in Real Projects? Keep high-priority student tasks under flat hierarchies rather than deep nested submenus.

Activity 4: Fix Wrong Structure

Audit the sitemap. Drag highlighted misplaced pages into their correct home categories.
📖 Learning Recap: Activity 4 (Fix wrong structure)
What Usability Problems Were Found? Misgrouped links (results under profile, receipts under resources) mislead user discovery search paths.
Which Principle Was Learned? Structural heuristics audits and layout predictability.
How Can It Be Applied in Real Projects? Align file nesting strictly under expected parent categories to match student mental models.
